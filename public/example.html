Universal Music Theory Library

A TypeScript library for music theory computation — 12-TET, microtonal tunings, harmony analysis, set theory, and more. Zero runtime dependencies.

This demo uses Tone.js for audio and abcjs to illustrate how UMT integrates with other libraries.

1. Tuning Systems

tuning · interval · note · presets

Select a tuning system and a root note. UMT generates the scale steps and computes the frequency of each note using the tuning's interval ratios.

−9 = C4, 0 = A4

11. API Reference

  
All exported classes and key methods

  
All symbols are available via window.UMT when using the IIFE bundle, or as named ESM imports. Steps = steps from A4 = 0 in the tuning's coordinate system.


  
  

    
Tuning Systems

    

      
Class / PresetSignatureDescription

      
        
EDOnew EDO(stepsPerOctave)Equal division of the octave. 12-TET = new EDO(12).

        
JustIntonationnew JustIntonation(name, ratios)Ratios as [numerator, denominator] pairs.

        
CentTuningnew CentTuning(name, cents[])Arbitrary cents per step. Used for historical temperaments.

        
NonOctaveTuningnew NonOctaveTuning(name, steps, periodRatio)Non-octave period (e.g. Bohlen-Pierce: 13 steps, ratio 3).

        
TET12EDO(12)Standard 12-tone equal temperament preset.

        
TET24EDO(24)Quarter-tone tuning.

        
TET31EDO(31)31-TET meantone approximation.

        
PtolemaicJIJustIntonationPtolemaic intense diatonic scale ratios.

        
FiveLimitJIJustIntonation5-limit chromatic just intonation.

        
WerckmeisterIIICentTuningHistorical well temperament (Bach era).

        
BohlenPierceNonOctaveTuning13 steps to the tritave (3:1).

tuning.getStepFromStandard(s)→ numberMaps a 12-TET step to the closest step in this tuning.
