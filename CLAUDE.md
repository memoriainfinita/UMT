# Universal Music Theory Library

## What is this

A TypeScript music theory library (`UMT`) supporting standard 12-TET, microtonal tunings (JI, EDOs, Werckmeister, Bohlen-Pierce), and advanced theory (Neo-Riemannian, Set Theory, Key Detection). Comes with a Next.js interactive demo page.

The library compiles to a standalone IIFE (`public/umt.js`) usable in any vanilla JS/HTML project with no dependencies.

## Stack

- **Library**: TypeScript, compiled with esbuild → `public/umt.js`
- **Demo**: Next.js 15 (App Router), React 19, Tailwind 4, abcjs (sheet music)
- **Audio**: Web Audio API (custom synth in `lib/audio.ts`)
- **Deploy**: GitHub Pages via `.github/workflows/deploy.yml`

## Commands

```bash
npm run dev          # Start Next.js dev server
npm run build        # Build library (umt.js) + Next.js app
npm run build:umt    # Build only the library bundle
npm run lint         # ESLint
```

## Project structure

```

  types.ts           # Base types (Cents, Hertz, Ratio)
  interval.ts        # Interval math
  tuning.ts          # TuningSystem, EDO, JustIntonation, CentTuning, NonOctaveTuning
  note.ts            # Note class
  scale.ts           # Scale class with modes
  chord.ts           # Chord class with voicings, inversions, voice leading
  dictionaries.ts    # CHORD_FORMULAS and SCALE_PATTERNS lookup tables
  parser.ts          # parseChordSymbol, parseScaleSymbol, parseNote, parseRomanProgression
  presets.ts         # Pre-built tuning systems, scales, chords
  harmony.ts         # Voice leading, chord detection, negative harmony, cadences
  circle.ts          # CircleOfFifths
  set-theory.ts      # Normal/prime form, interval vector
  key-detection.ts   # Krumhansl-Schmuckler algorithm
  neo-riemannian.ts  # PLR transformations
  scala.ts           # Scala (.scl) file parser
  rhythm.ts          # Duration, TimeSignature, Polyrhythm, Euclidean
  stream.ts          # MusicStream event container
  abc-bridge.ts      # Converts UMT objects to ABC notation
  index.ts           # Re-exports everything
  umt.ts             # IIFE entry point for the bundle

app/page.tsx         # Demo page (~1300 lines, single component)

  SheetMusic.tsx     # abcjs wrapper component
public/
  umt.js             # Compiled bundle (generated by build:umt)
  example.html       # Vanilla JS demo
```

## Coordinate system

All pitch positions are expressed as **steps from A4 = 0** within a `TuningSystem`. A4 = 440Hz = step 0. C4 = step -9 in 12-TET.

## Key conventions

- `TuningSystem` is abstract. Always use `EDO`, `JustIntonation`, `CentTuning`, or `NonOctaveTuning`.
- `Chord.smoothTransition` handles voice leading automatically — use it for progressions.
- `parseRomanProgression` supports: `ii7`, `V7`, `V7alt`, `V7/ii` (applied), `subV7` (tritone sub).
- `ScalaTuning` (from `.scl` files) requires the last entry to be the octave (e.g., `2/1` or `1200.0`).
- Note names in the library are English (no Spanish strings in library code).
